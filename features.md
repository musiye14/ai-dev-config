# 功能列表(工具无关的开发习惯)

> 这是"我要什么习惯"的清单,工具无关。部署到任意 Coding Agent 时,把本文件内容注入该工具的 instructions 文件(如 `~/.config/opencode/AGENTS.md`、`~/.claude/CLAUDE.md`、`~/.codex/AGENTS.md`)。
>
> 加新习惯 = 在这里加一节。删习惯 = 删一节。

## 1. 对话时间戳(硬要求)

每次完成一轮回复时,在该回复**正文第一行**输出时间戳徽章:

```
⏱ <HH:MM:SS> · prev <N>s
```

- `<HH:MM:SS>`:你完成本条回复的本地时间(24 小时制)。
- `<N>`:本轮耗时秒数(整数)= 从用户发出本轮首个 prompt 到你完成回复的墙钟时长。
- 时间戳行与回复正文之间空一行。

**例外**:单轮耗时 <2s 时不输出;用户说"安静"/"不输出时间"时本会话停用。

**为什么靠 AI 自输出**:TUI 是编译二进制,无法在消息流插入时间戳。AI 自输出是唯一跨工具一致的实现。

## 2. 子任务显示

- 工具原生显示活跃子任务数 + 单个持续时间(如 Claude statusLine)→ 无需额外动作。
- 工具不原生显示(Codex / opencode)且本轮**确实触发过**子任务 → 回复末尾加一行:`🧵 子任务:活跃 X / 累计 Y · 最长 Zs`。
- 未触发子任务不输出。不编造数字。

## 3. 状态行

- 工具原生支持可脚本化状态行(Claude `statusLine`)→ 由本地脚本渲染 `[$model] 📁 $dir | 🧵 X 活跃 | ⏱ last Ns | P% · $cost`,AI 不干预。
- 工具无状态行(Codex / opencode)→ 放弃,不硬注入终端标题。

## 4. 系统通知

对话轮结束、需确认操作时,由工具原生通知或本地脚本弹系统通知(见 registry 的通知脚本)。AI **无需**调用工具发通知。

## 5. Skill 优先

有 skill 清单时,用户请求匹配某 skill 描述 → 优先调用 `skill` 工具加载,而不是凭记忆回答。

---

## 部署到新工具

1. 把本文件(§1-§5)内容写入该工具的 instructions 文件。工具无关习惯立即生效。
2. 读 `registry.md`,逐条按各工具的安装方式部署插件/skill(agent 看 GitHub 自己装,不搬冗余文件)。
3. 探查工具能力后落地:原生能做→配原生;做不到但 AI 能自补偿(如时间戳)→靠本规则;完全做不到→豁免,不硬塞降级。
4. 用户确认可靠 → 存快照到 `cache/<tool>/<ver>/`。下次同版本命中直接复制。
